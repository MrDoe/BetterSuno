# BetterSuno: Project Context for GitHub Copilot Agents

BetterSuno is a browser extension (v3) built for Chrome and Firefox that enhances the [suno.com](https://suno.com) experience. It provides real-time notifications, advanced song library management, batch downloading, offline caching, and an integrated mini-player.

## Architecture Overview

The project uses a multi-process messaging architecture with persistent state management via IndexedDB.

- **Content Scripts**: 
    - [content.js](content.js) handles the primary UI (notification panel, settings tabs).
    - [downloader.js](downloader.js) manages library filtering, playback controls, and batch downloading.
- **Injected Scripts**:
    - [content-fetcher.js](content-fetcher.js) runs in the page context to access Suno's internal APIs and authentication tokens.
- **Background/Service Worker**:
    - [background.js](background.js) coordinates state, routes messages, and manages downloads.
- **Offscreen (Chrome-only)**:
    - [offscreen.js](offscreen.js) / [offscreen.html](offscreen.html) provide persistent polling for notifications and library updates.
- **Storage**:
    - [idb-store.js](idb-store.js) provides an IndexedDB abstraction for caching songs (MP3/WAV), user preferences, and notification state.

## Key Features for Agents to Know

- **Polling**: Always-on polling (default 2 mins) via offscreen document or persistent background script.
- **Offline Cache**: Users can download/cache songs to a local IndexedDB `audioCache` for offline playback within the extension.
- **Batch Operations**: Supports downloading multiple songs, metadata (lyrics), and cover art simultaneously.
- **Cross-Browser Build**: A custom [build.js](build.js) script handles platform-specific manifest adjustments and file structures for Chrome and Firefox.
- **Permissions**: Requires host permissions for `https://suno.com/*`, `https://*.suno.com/*`, and `https://studio-api.prod.suno.com/*`.

## Development Workflow

- **Build**: Use `node build.js` to generate `dist/chrome` and `dist/firefox`.
- **Tasks**: VS Code tasks are available for building, deploying, and packaging (`Build All`, `Launch Firefox with Extension`, etc.).
- **Manifest**: The root [manifest.json](manifest.json) is the template used by [build.js](build.js) to generate browser-specific manifests.

## Technical Constraints

- **Manifest V3**: Strictly follows MV3 patterns (Service Workers, Offscreen Documents).
- **CORS**: Interacts with Suno APIs; requires credentials and specific origin headers.
- **Performance**: Uses `IntersectionObserver` in content scripts to handle large song libraries without UI lag.
